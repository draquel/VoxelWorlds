VoxelWorlds

Engine architecture · Unreal Engine 5.8 · C++ + compute shaders

Eleven modules that turn noise into worlds.

A GPU-driven voxel terrain engine — procedural generation, three meshing algorithms, watertight LOD streaming, and hybrid rendering. Every capability lives in a small module that does one job and cooperates with the rest through interfaces, never direct coupling.

Fig. B — Chunk lifecycle

From viewer to visible terrain

The path a single chunk travels every time the camera moves far enough to change what's on screen. The coordinator (VoxelStreaming) drives it; the three green stages run on the GPU, each with a CPU fallback.

01 Camera moves Viewer crosses a chunk boundary and the streaming tick wakes up. VoxelStreaming
02 Pick chunks Distance bands choose the load / unload set and a LOD level per chunk. VoxelLOD
GPU · CPU fallback 03 Generate voxels Noise → world mode → biome → caves → ore → water flags, into a packed FVoxelData grid. VoxelGeneration
04 Apply edits Sparse add / subtract / paint overlay is merged over the generated field. VoxelCore · EditLayer
GPU · CPU fallback 05 Mesh Cubic, Marching Cubes, or Dual Contouring — plus a watertight LOD-seam weld at chunk borders. VoxelMeshing
GPU · CPU fallback 06 Render + collide GPU-driven custom vertex factory (or PMC), while collision cooks async on Chaos. VoxelRendering + Streaming
07 Decorate Foliage & trees, water tiles, minimap, and runtime PCG scatter fill the chunk in. Scatter · Map · PCG

Fig. C — Capability map

What the engine ships

The shipping feature surface, grouped by system. Colour follows the same layer coding as the module map.

Meshing

  • Cubic — greedy, face-culled, ambient occlusion
  • Marching Cubes — smooth isosurface + geomorph seams
  • Dual Contouring — sharp features via QEF solver
  • Each with a CPU and GPU compute path

Generation

  • CPU / GPU noise — Perlin, Simplex, Cellular, Voronoi
  • FBM fractals + domain warping
  • Caves — cheese, spaghetti, noodle layers
  • Ore veins & column-scan water fill

World modes

  • Infinite plane — heightfield, extends forever
  • Island / bowl — radial falloff, ocean level
  • Spherical planet — radius-based, curved
  • Swappable via IVoxelWorldMode

LOD & seams

  • Pluggable strategy via IVoxelLODStrategy
  • Distance bands + refine/coarsen hysteresis
  • Vertex-shader LOD morphing
  • Watertight seams (GPU ↔ CPU parity)

Rendering

  • GPU-driven custom vertex factory
  • ProceduralMeshComponent fallback for tools
  • Runtime Virtual Texture integration
  • Second-pass water tile rendering

Streaming

  • Distance-based load / unload queues
  • Async mesh generation + stale-cull
  • Async Chaos collision cooking
  • Thrash instrumentation + benchmark

Editing

  • Sparse edit-layer overlay
  • Add / subtract / paint brushes
  • Discrete block editing + undo/redo
  • Binary persistence

Materials

  • Material registry — up to 256 types
  • Texture-array atlas (triplanar / cubic)
  • Unified M_VoxelMaster
  • Per-quad materials at borders

Scatter & foliage

  • Biome-driven placement
  • HISM / ISM instanced rendering
  • Deterministic cross-chunk tree injection
  • Edit-aware cleanup

Map

  • Map / minimap subsystem
  • Samples world height & biome
  • Blueprint-exposed queries

PCG decoration

  • Runtime PCG scatter over voxel surface
  • Per-biome graph dispatcher
  • Edit-aware surface sampler
  • Isolated behind the PCG plugin

Editor tools

  • Dockable field-visualization panel
  • 8 built-in sampled fields
  • Field preview actor in-viewport
  • Stripped from shipping builds
