VoxelWorlds

Engine architecture · Unreal Engine 5.8 · C++ + compute shaders

Eleven modules that turn noise into worlds.

A GPU-driven voxel terrain engine — procedural generation, three meshing algorithms, watertight LOD streaming, and hybrid rendering. Small modules, each doing one job, cooperating through interfaces rather than direct coupling.

Chunk lifecycle

From viewer to visible terrain

03 / 05 · green stages run on the GPU
01 Camera moves Viewer crosses a chunk boundary; the streaming tick wakes up. VoxelStreaming
02 Pick chunks Distance bands choose the load/unload set and a LOD per chunk. VoxelLOD
GPU · CPU fallback 03 Generate voxels Noise → world mode → biome → caves → ore → water, into a packed FVoxelData grid. VoxelGeneration
04 Apply edits Sparse add/subtract/paint overlay merges over the generated field. VoxelCore · EditLayer
GPU · CPU fallback 05 Mesh Cubic, Marching Cubes, or Dual Contouring — plus a watertight LOD-seam weld. VoxelMeshing
GPU · CPU fallback 06 Render + collide GPU-driven vertex factory (or PMC); collision cooks async on Chaos. Rendering + Streaming
07 Decorate Foliage & trees, water tiles, minimap, and runtime PCG scatter. Scatter · Map · PCG

Why the shape matters

Small modules, clean seams

05 / 05

Layered like the terrain it generates — bedrock at the bottom, surface systems on top, and nothing ever reaches upward.

01
Swappable by interface

Change a mesher or LOD strategy without touching the core — everything integrates through interfaces, not concrete types.

02
GPU-first, all the way to the screen

Generate, mesh, and render on the GPU; the CPU paths exist as fallbacks, not the hot path.

03
One coordinator, not a web

VoxelStreaming is the single hub that wires the domain systems together — the rest stay decoupled.

04
Ship simple, optimize later

Distance-band LOD and a thin facade today; quadtree/octree and more world modes slot in without a rewrite.
